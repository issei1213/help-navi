# Implementation Plan

- [x] 1. Docker ComposeにPostgreSQLサービスを追加し、環境変数テンプレートを整備する
- [x] 1.1 Docker Compose設定にPostgreSQLサービスを定義する
  - 既存の`docker-compose.yml`にPostgreSQL 17サービスを追加し、MinIOサービスと並列で動作するよう構成する
  - `postgres:17`公式イメージを使用し、開発用のデフォルト認証情報（ユーザー名・パスワード・データベース名）を環境変数で設定する
  - `POSTGRES_DB`環境変数によりデフォルトデータベースがコンテナ初回起動時に自動作成されるようにする
  - ホストポート5432をコンテナの5432にマッピングし、MinIO(9000/9001)やNext.js(3000)とポート競合しないことを確認する
  - `postgres-data`Dockerボリュームを定義してデータ永続化を実現し、既存の`minio-data`ボリュームと共存させる
  - `pg_isready`コマンドによるヘルスチェックを設定する（interval: 5s, timeout: 5s, retries: 5, start_period: 10s）
  - 既存MinIOサービスの起動・停止に影響を与えないことを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 2.3, 5.3, 5.4_

- [x] 1.2 (P) `.env.example`にPostgreSQL接続用環境変数テンプレートを追加する
  - 既存S3セクションの直後に「PostgreSQLデータベース設定」セクションを追加する
  - `DATABASE_URL`接続文字列と個別環境変数（ホスト、ポート、ユーザー名、パスワード、データベース名）のテンプレートを定義する
  - 既存S3環境変数と同じコメントスタイル・セクション区切りに従う
  - 開発環境のデフォルト値を記入し、コピーするだけで動作する状態にする
  - Docker Compose設定のデフォルト認証情報と値が整合していることを確認する
  - _Requirements: 2.1, 2.2, 2.3, 2.4_

- [x] 2. Prisma ORMを導入し、スキーマファイルとクライアントモジュールを構築する
- [x] 2.1 Prisma ORMパッケージをインストールしスキーマファイルを配置する
  - `prisma`（CLI）と`@prisma/client`（ランタイム）のPrisma 6.19.x系パッケージをプロジェクトに追加する
  - `prisma/schema.prisma`にdatasource（PostgreSQL、`DATABASE_URL`環境変数参照）とgenerator（`prisma-client-js`）を定義する
  - モデル定義は空の初期状態とし、今回はテーブル定義を含めない
  - `prisma validate`でスキーマの構文が正しいことを確認する
  - _Requirements: 3.1, 3.2, 6.1, 6.6_

- [x] 2.2 シングルトンパターンでPrismaクライアントモジュールを実装する
  - インフラストラクチャ層に既存S3クライアントと同じ構成パターンでPrismaクライアントモジュールを配置する
  - `DATABASE_URL`環境変数から接続情報を取得してPostgreSQLへの接続を確立する
  - `globalThis`を使用したシングルトンパターンにより、開発時のホットリロードによる接続プール枯渇を防止する
  - 開発環境ではクエリログ・エラー・警告を出力し、本番環境ではエラーのみのログレベルとする
  - データベース接続失敗時に、接続先情報と対処方法を含む日本語エラーメッセージを出力する
  - JSDocコメントを日本語で付与し、モジュールの用途と注意事項を記述する
  - _Requirements: 3.3, 3.4, 3.5, 3.6_
