# Implementation Plan

- [x] 1. ツール表示名のユーティリティを作成する
  - 既知のツール名（webSearch, s3ListObjects, s3GetObject, s3PutObject）を日本語表示名にマッピングする関数を実装する
  - 未登録のツール名が渡された場合はそのまま返却する
  - ユニットテストで既知・未知のツール名の変換を検証する
  - _Requirements: 1.3_

- [x] 2. ツール状態アイコンコンポーネントを作成する
- [x] 2.1 (P) ツールの実行状態に応じたアイコンとアニメーションを表示するコンポーネントを実装する
  - 実行中状態（input-streaming, input-available）では回転するスピナーアイコンを表示する
  - 完了状態（output-available）では緑色のチェックマークアイコンを表示する
  - エラー状態（output-error）では赤色のエラーアイコンを表示する
  - アイコンはSVGインラインで実装し、外部ライブラリに依存しない
  - _Requirements: 1.4, 3.2, 5.1_

- [x] 2.2 (P) ツール入出力の詳細表示コンポーネントを実装する
  - 入力パラメータをkey-value形式で構造化表示する
  - 実行成功時は出力結果をJSON整形して表示する
  - エラー時はエラー情報を赤色背景で視覚的に区別して表示する
  - 出力が500文字を超える場合は切り詰めて省略記号を付加する
  - _Requirements: 2.2, 5.2, 5.3_

- [x] 3. ツール呼び出しパーツコンポーネントを作成する
  - ツール名の表示、状態アイコン、展開/折りたたみ機能を統合した個別ツール呼び出し表示を実装する
  - デフォルトでは折りたたまれた状態で表示し、クリックで展開・折りたたみを切り替える
  - 実行中状態（input-streaming, input-available）ではデフォルト展開状態にする
  - 実行中から完了・エラーに状態が遷移した際に自動的に折りたたむ
  - CSSのtransitionとmin-heightでレイアウト安定性を確保し、ちらつきやジャンプを抑制する
  - ツール名が不正（undefined等）の場合はレンダリングをスキップし、コンソールに警告を出力する
  - _Requirements: 1.1, 1.2, 1.3, 2.1, 2.2, 2.3, 2.4, 4.2_

- [x] 4. MessageBubbleのパーツレンダリングを拡張する
  - 現在のテキストパーツのみの抽出ロジックを廃止し、partsの配列を順序通りにイテレートする方式に変更する
  - テキストパーツは既存のMarkdownレンダリングに委譲し、ツール呼び出しパーツはツール呼び出しパーツコンポーネントに委譲する
  - 複数のツール呼び出しが含まれる場合は実行順にリスト表示する
  - ツールパーツとテキストパーツが交互に出現する場合も正しい順序で描画する
  - 各ツール呼び出しの実行ステータスをアイコンで表示する
  - 全ツール完了後にテキスト応答をストリーミング表示する動作を維持する
  - ユーザーメッセージ（role=user）は既存のテキスト表示を維持する
  - _Requirements: 3.1, 3.2, 3.3, 4.1_

- [x] 5. MessageListの自動スクロールをツールパーツに対応させる
  - ツールパーツの追加や状態更新時にも自動スクロールを発動させる
  - メッセージの最後のパーツ数の変化を監視対象に含め、パーツ追加時にスクロールを発火する
  - ストリーミング中にツール呼び出しが含まれるメッセージでも最新の状態が画面内に表示されるようにする
  - _Requirements: 4.3_

- [x] 6. 統合テストで全体の動作を検証する
- [x] 6.1 MessageBubbleの統合テストを実装する
  - ツールパーツとテキストパーツが混在するメッセージで正しいレンダリング順序を検証する
  - 複数ツール呼び出しを含むメッセージのリスト表示を検証する
  - エラー状態のツールが赤色インジケーターで表示されることを検証する
  - _Requirements: 3.1, 3.3, 4.1, 5.1_

- [x]* 6.2 個別コンポーネントの受け入れ基準テストを実装する
  - ToolInvocationPartの各状態（input-streaming, input-available, output-available, output-error）での正しい描画を検証する
  - 展開/折りたたみのクリック動作と初期状態を検証する
  - ToolDetailViewの入力/出力/エラー表示と長大データの切り詰めを検証する
  - _Requirements: 1.1, 1.2, 1.4, 2.1, 2.2, 2.3, 2.4, 4.2, 5.2, 5.3_
