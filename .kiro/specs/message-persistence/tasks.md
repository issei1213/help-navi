# Implementation Plan

- [ ] 1. LLMによる会話タイトル自動要約生成機能を実装する
- [x] 1.1 (P) タイトル生成関数を作成し、Mastraエージェント経由でユーザーメッセージを30文字以内の会話タイトルに要約する
  - 既存の `sample-agent` を使用した非ストリーミング呼び出し（`agent.generate()`）でタイトルを生成する
  - プロンプトで「30文字以内の簡潔な会話タイトル」を指示し、レスポンスをトリムして最大文字数に切り詰める
  - LLM呼び出し失敗時はフォールバックとしてユーザーメッセージの先頭30文字を返却する
  - 空のレスポンスが返却された場合もフォールバックを適用する
  - _Requirements: 3.1, 3.3_

- [x] 1.2 タイトル生成関数をユーザーメッセージ保存処理に統合する
  - 会話タイトルがデフォルト値「新しいチャット」であり、かつメッセージ数が0-1件の場合のみタイトル生成を呼び出す
  - タイトルがデフォルト以外に変更済みの場合は生成をスキップする
  - タイトル生成は非同期で実行し、チャットストリーミング応答をブロックしない（awaitせずにPromiseを発行）
  - タイトル生成の失敗がメッセージ保存やストリーミングに影響しないよう、独立したtry-catchで保護する
  - _Requirements: 3.1, 3.2, 3.3, 3.4_

- [x] 2. ストリーム完了時のAI応答メッセージ永続化を接続する
  - 既に定義済みの `saveAssistantMessage` 関数をストリーム完了コールバックに接続する
  - `handleChatStream` / `createUIMessageStreamResponse` のストリーム完了を検知する仕組みを実装する（ストリームラッピングまたはonFinishコールバック方式を検証し適切な方式を採用）
  - `conversationId` が指定されている場合のみAI応答を保存し、未指定の場合はスキップする（後方互換性維持）
  - AI応答保存の失敗がストリーミング応答に影響しないよう、エラーログ出力のみで処理を継続する
  - _Requirements: 2.2, 2.4, 6.3, 7.1, 7.2_

- [ ] 3. 既存実装の要件充足を検証し、不足箇所を修正する
- [x] 3.1 (P) 会話セッションCRUDの要件適合を検証する
  - 新規会話作成時にデフォルトタイトル「新しいチャット」が設定され、一意のIDが返却されることを確認する
  - 会話一覧が更新日時降順で最新50件返却されること、各項目にID・タイトル・更新日時が含まれることを確認する
  - タイトル更新時にトリム処理が適用されること、空文字・スペースのみのタイトルで400エラーが返却されることを確認する
  - 会話削除時にメッセージがカスケード削除されることを確認する
  - 存在しない会話IDへの操作で404エラーが返却されることを確認する
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6_

- [x] 3.2 (P) メッセージ保存・取得の要件適合を検証する
  - ユーザーメッセージ保存時に一意のID、ロール、コンテンツ、作成日時、会話IDが保持されることを確認する
  - メッセージ取得時に作成日時昇順で返却されること、各フィールド（ID、ロール、コンテンツ、作成日時）が含まれることを確認する
  - メッセージが存在しない会話で空配列が返却されることを確認する
  - _Requirements: 2.1, 2.3, 4.1, 4.2, 4.4_

- [x] 3.3 (P) エラーハンドリングとHTTPステータスコードの要件適合を検証する
  - 全APIエンドポイントでDB接続失敗時に500ステータスコードとエラーメッセージが返却されることを確認する
  - 不正なリクエストボディ（messagesフィールド欠損、空配列）で400ステータスコードが返却されることを確認する
  - 全エンドポイントが適切なHTTPステータスコード（200/201/400/404/500）を返却していることを確認する
  - _Requirements: 6.1, 6.2, 6.4_

- [x] 3.4 (P) フロントエンド統合（会話復元・UIレンダリング）の要件適合を検証する
  - サイドバーから既存会話を選択した際にメッセージ履歴がUIMessage形式に変換されてチャットUIに表示されることを確認する
  - ユーザーメッセージとAI応答メッセージがそれぞれ適切にレンダリングされることを確認する
  - `conversationId` が未指定の場合にメッセージ永続化がスキップされ、既存ストリーミング動作のみが実行されることを確認する
  - `conversationId` が指定されている場合にメッセージ永続化とストリーミング応答が併せて実行されることを確認する
  - _Requirements: 4.3, 7.1, 7.2_

- [x] 4. データベーススキーマの要件適合を検証する
  - Conversationテーブルに必須フィールド（ID: CUID、タイトル、作成日時、更新日時）が定義されていることを確認する
  - Messageテーブルに必須フィールド（ID: CUID、ロール、コンテンツ、作成日時、会話ID: 外部キー）が定義されていることを確認する
  - `conversationId` カラムにインデックスが設定されていることを確認する
  - カスケード削除制約（`onDelete: Cascade`）が設定されていることを確認する
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

- [ ] 5. 結合テストでエンドツーエンドの永続化フローを検証する
- [x] 5.1 会話APIエンドポイントの結合テストを作成する
  - POST /api/conversations による新規会話作成（デフォルトタイトル、201ステータス）をテストする
  - GET /api/conversations による会話一覧取得（降順50件制限）をテストする
  - PATCH /api/conversations/[id] によるタイトル更新（正常系・空タイトル拒否・404エラー）をテストする
  - DELETE /api/conversations/[id] によるカスケード削除（正常系・404エラー）をテストする
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 6.1, 6.2, 6.4_

- [x] 5.2 メッセージAPIエンドポイントの結合テストを作成する
  - GET /api/conversations/[id]/messages によるメッセージ取得（昇順・フィールド検証・空配列・404エラー）をテストする
  - POST /api/chat でconversationId指定時のユーザーメッセージ保存をテストする
  - POST /api/chat でconversationId未指定時の後方互換動作をテストする
  - LLMタイトル自動生成（正常系・フォールバック）をテストする
  - _Requirements: 2.1, 2.2, 2.4, 3.1, 3.3, 4.1, 4.2, 4.4, 6.3, 7.1, 7.2_

- [x]* 5.3 (P) フロントエンドフックのユニットテストでアクセプタンスクライテリアを網羅する
  - `useConversations` フックの会話CRUD操作（作成・選択・更新・削除）と楽観的キャッシュ更新をテストする
  - `useChatSession` フックのTransport生成（conversationId連動）、sendMessage無効化（conversationId: null時）をテストする
  - エラーハンドリング（APIフェッチ失敗時のフォールバック動作）をテストする
  - _Requirements: 1.1, 1.2, 1.3, 1.5, 4.1, 7.1, 7.2_
